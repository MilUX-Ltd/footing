# {{title}} — Mobilisation Checklist

What needs to be true before this engagement moves from `scoping/` to `active/`. The list is binary: each item is either done or it isn't. When everything is ticked, the user makes the call to promote the engagement (marked by a contract or a confirmed start date), not the checklist.

## Items

- [x] Customer organisation page exists and is current ({{customer_wikilink}}).
- [x] Engagement folder scaffolded in `Customer Engagements/scoping/{{folder_name}}/`.
- [x] Tracking card linked or created: {{tracking_card_url}}.
- [x] Stakeholder map skeleton seeded from CRM.
- [x] RAID log seeded with default entries.
- [ ] Outcome statement confirmed with customer.
- [ ] Information Architecture table filled (external storage paths, accounting reference, data classification confirmed).
- [ ] Stakeholder map sharpened with customer input.
- [ ] Comms plan drafted and agreed.
- [ ] Scope, objectives, and metrics confirmed in writing.
- [ ] Cadence (reporting + meetings) agreed.
- [ ] Roles and responsibilities confirmed.
- [ ] NDA in place (where required).
- [ ] Sector-specific compliance evidence shared (where required).
- [ ] Day rate or fixed price agreed.
- [ ] Invoicing schedule agreed (monthly arrears, milestone-linked, retainer up front, etc.).
- [ ] Change control mechanism agreed and recorded on the index.
- [ ] Capacity and availability statement agreed and recorded on the index.
- [ ] Data classification for this engagement agreed.
- [ ] Scope and Deliverables table populated with acceptance criteria and sign-off owners.
- [ ] Engagement-level Definition of Done written.
- [ ] Reporting and governance cadence agreed and entered on the comms plan.
- [ ] Benefits measurement table populated on the outcome page.
- [ ] PO or contract reference captured.
- [ ] Contract signed.
- [ ] Start date set.

## Notes

Capture any context-specific items here. Sectors and customer types add bespoke checks (export control, vetting, security clearance, regulatory access, etc.).

## When ready to promote

When the bottom items (contract signed, start date set) land, ask your agent: "promote the {{folder_name}} engagement to active". A separate skill handles the folder move, status update, wikilink rewrite, and tracking card column shift. Promotion is a manual call, not an automatic consequence of the checklist filling.
